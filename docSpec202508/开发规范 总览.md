


通用开发规范分类
分类	内容简述
✅ 代码规范	命名规范、缩进风格、注释规范、异常处理、魔法值约束等
✅ 项目结构规范	包结构、模块划分、层次设计（Controller/Service/DAO）
✅ API 设计规范	路由风格、请求/响应结构、状态码定义、错误信息格式化
✅ 日志规范	日志等级、结构化日志、traceId、日志字段统一、脱敏处理
✅ 异常处理规范	异常统一捕获、错误码枚举、异常链路追踪、用户友好提示
✅ 安全规范	参数校验、鉴权机制、防重放、防注入、加密传输、安全审计
✅ 数据库规范	命名规范、字段类型、主键设计、索引策略、DDL/DML 版本控制
✅ 测试规范	单元测试、接口测试、集成测试、代码覆盖率、测试命名与分层
✅ 注释与文档规范	类/方法注释、接口文档、设计说明、变更日志、README模板
✅ DevOps/CI 规范	分支管理、提交规范（如 Conventional Commits）、发布流程
✅ 配置管理规范	YAML/JSON配置、分环境配置、动态配置中心、安全参数隔离
✅ 审计与日志追踪规范	操作日志、登录日志、重要事件日志、字段变更记录、接口审计
